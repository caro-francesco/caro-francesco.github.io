un processo è un programma mandato in esecuzione, il thread è un flusso di esecuzione all'interno di un processo. il context-switch è il passaggio di esecuzione tra i thread, il S.O salva i registri del thread nella thread-table e carica quelli del thread successivo. è più veloce del context-switch tra processi in quanto i thread condividono certe cose.

A sinistra un immagine raffigurativa dei vari stati dei thread.

thread_create: chiede la creazione di un thread
thread_exit: thread termina se stesso
thread_join: un thread sincronizza la sua fine con la fine di un altro thread
thread_yeld: rilascio cpu ad un altro thread in coda
thread_sleep: blocca il thread per un certo tempo

abbiamo thread foreground: quando termina il thread principale aspetta la fine degli altri thread
abbiamo thread background: quanto termina il thread principale termina gli altri thread

thread a livello utente: molti a uno
deve essere supportato dal S.O.
vantaggi: non esegue la trap
svantaggi: se un thread va in attesa tutto il proceso va in attesa

thread a livello kernel: uno a uno
usa le strutture dati nel kernel, viene gestita direttamente dal S.O, le funzioni sono tutte trap
vantaggi: se viene bloccato un thread gli altri non vengono bloccati
svantaggi: context-switch piu lento
